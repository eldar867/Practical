# Practical
Предметная область
Система оформления заказа кофе в кофейне. Пользователь настраивает напиток: база, размер, молоко, сиропы, сахар, лед. 

Ограничения

Язык: Python 3.10+.
Внешние зависимости запрещены (только стандартная библиотека).
Вся реализация в одном файле main.py.
Что нужно реализовать

Модель заказа
Класс CoffeeOrder
Поля:
base: одно из ["espresso", "americano", "latte", "cappuccino"], не может быть пустым, если пустое выкинуть исключение.
size: одно из ["small", "medium", "large"], не может быть пустым, если пустое выкинуть исключение.
milk: одно из ["none", "whole", "skim", "oat", "soy"], по умолчанию "none".
syrups: неизменяемая коллекция (tuple[str, ...]), по умолчанию пусто, может быть несколько значений
sugar: int, 0 по умолчанию.
iced: bool, False по умолчанию.
price: float, должно подсчитываться в самом конце.
description: str, человекочитаемое описание заказа, как должно выглядеть описано ниже.
Требования:
Метод str возвращает description (или краткую строку с ценой, если description пуст).
Базовые цены:
espresso: 200
americano: 250
latte: 300
cappuccino: 320
Множители размера:
small: 1.0, medium: 1.2, large: 1.4
Доплаты:
молоко: whole/skim = 30, oat = 60, soy = 50, none = 0.0
сироп: 40 за каждый
лед: 0.2 фиксировано при iced=True
Лимиты:
сахар: 0-5
сиропов: максимум 4
Расчет цены:
формулу нужно продумать самостоятельно
Описание заказа:
Строка формата: "<size> <base /> [with <milk> milk] [+syrup list] [(iced)] [<sugar> tsp sugar]"
Элементы, равные значению по умолчанию, в описание не включать.
После кода.

Базовый заказ: задать base, size, несколько опций; проверить:
правильность типов, непустую цену, наличие опций в объекте заказа
Стиль и качество

Аннотации типов обязательны.
Константы цен и лимитов собрать в одном месте (как атрибуты класса).
Именование методов и переменных — понятное и единообразное.
Создать репозиторий, ссылку на репозиторий скинуть как ответ на это задание.
